Compiler is a class which manages view compilation, from any source.

It is a built-in for pluggy, not yet decoupled enough to be considered replaceable.

Examples:

Defining a new compiler

require 'erb'

Compiler.new(:erb) do |text, b|
  ERB.new(text).result(b)
end

Compiling text with ERB

Compiler.erb("<%= 1+1 %>") #=> "2"

Compiling an ERB file

# foo.html.erb
<h1>I have a random number for you: <%= rand(0,10) %></h1>

# File in same directory
filepath = File.join(Dir.pwd, 'foo.html.erb')
Compiler.compile(filepath) #=> "2"

Instance Method Details

#run(text, b = @block.binding) ⇒ Object

Blocks are called in the context that they are created in and are responsible for managing their own bindings. They should not expect to have instance variables available to them that are in the binding they are passed.

Parameters:

  • text —

    The value to be compiled by the compiler. Typically it is a String, but it can be anything, so long as the compiler @block can handle it.

  • b (Binding) (defaults to: @block.binding) —

    Pass a binding which is also forewarded to the @block.
